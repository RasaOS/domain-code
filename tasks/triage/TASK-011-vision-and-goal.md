# TASK-011 — Project vision & goal concept

**User story.** As a project lead, I want the project to hold an
explicit long-term **vision** and a current short-term **goal**, so
every session knows where the project is headed and what it's
working toward right now.

**Why.** Phases and tasks describe *what* and *how* but not *why*
or *toward what*; an explicit vision + goal keeps long-term
direction and the immediate objective visible at all times.

**Notes (raw idea).** A mission/goal/vision concept. The **vision**
is long-term — "how does the user envision this project long
term?" The **goal** is a short-term action to achieve. So we always
know what our goal is along the way, what the vision is, etc. Open
questions: where it lives (a `VISION.md` / `GOAL.md`, or fields in
`CLAUDE.md`); whether the goal is single-valued and rotated as it's
hit; surfaced at session start.
